## PART 3: CLEAN CODE PRINCIPLES

### Meaningful Names
- Use intention-revealing names
- Avoid magic numbers
- Name constants meaningfully
- Classes: nouns
- Methods: verbs

### Functions
- Small (20 lines max)
- Do one thing well
- Few parameters (max 3)
- No side effects
- DRY (Don't Repeat Yourself)

### Formatting
- Vertical spacing
- Horizontal alignment
- Team rules matter most

### Error Handling
- Use exceptions
- Don't return null
- Don't pass null
- Provide context in errors

### Unit Testing
- Test-driven development (TDD)
- AAA: Arrange, Act, Assert
- One assert per test
- Fast, Isolated, Repeatable

### Classes
- SRP: Single Responsibility
- Open/Closed Principle
- Dependency Injection

## PART 7: CODING RULES

### Always Do
1. Write readable code
2. Comment WHY, not WHAT
3. Test your code
4. Use version control
5. Keep functions small
6. Name variables clearly

### Never Do
1. Don't repeat code
2. Don't use magic numbers
3. Don't ignore errors
4. Don't write clever code
5. Don't skip tests

## PART 8: BEST PRACTICES

### Code Review Checklist
- [ ] Does it work?
- [ ] Is it readable?
- [ ] Are there tests?
- [ ] Any security issues?
- [ ] Performance concerns?

### Debugging Steps
1. Read the error
2. Check the line number
3. Use console.log
4. Google the error
5. Ask for help

1. HTML/CSS basics
2. JavaScript fundamentals
3. Build simple projects
4. Learn a framework
